## 1. Epic / Feature / Story 分層

### Epic-1：數位基礎建設資訊入口（對應 PRD F-01~F-04）
- Feature-1.1：首頁主視覺
  - Story-1.1.1：作為一名決策者，我希望能在首頁快速了解三大核心與平台宗旨，以便掌握產業全貌。
- Feature-1.2：導覽列
  - Story-1.2.1：作為一名用戶，我希望能透過導覽列切換不同主題區塊，以便快速進入所需內容。
- Feature-1.3：主題/語言切換
  - Story-1.3.1：作為一名開發者，我希望能切換深色/淺色主題與中英語言，以便提升瀏覽體驗。

### Epic-2：產業資訊聚合（對應 PRD F-03, F-06~F-08）
- Feature-2.1：最新消息
  - Story-2.1.1：作為一名用戶，我希望能看到最新產業消息，以便掌握動態。
- Feature-2.2：政府資源
  - Story-2.2.1：作為一名業者，我希望能查詢政府資源內容，以便獲取政策與補助資訊。
- Feature-2.3：業者地圖
  - Story-2.3.1：作為一名業者，我希望能搜尋與篩選同業資訊，以便尋找合作或競爭對手。
- Feature-2.4：數位實證案例
  - Story-2.4.1：作為一名用戶，我希望能瀏覽優良專案案例，以便參考學習。

### Epic-3：平台可用性與合規（對應 PRD F-09, F-10）
- Feature-3.1：無障礙支援
  - Story-3.1.1：作為一名身心障礙者，我希望網站能符合無障礙標準，以便順利瀏覽。
- Feature-3.2：外部平台連結
  - Story-3.2.1：作為一名用戶，我希望能直接連結至 AI、GitHub、HuggingFace 平台，以便延伸應用。
